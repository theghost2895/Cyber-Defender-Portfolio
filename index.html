SOC Playbook
      

SOC Analyst Playbook

      

        A glimpse into how I think through common security scenarios—from first alert to final action.       

      
        
          
Case Study: Multiple Failed Logins from One IP
          

            Scenario: SIEM raises an alert for repeated failed logins targeting a privileged             account.           

          
                
  • Confirm: Validate the volume and timeframe of failed logins.
    •             
  • Context: Check geolocation, previous activity, and known devices.
    •             
  • Success? Look for any successful logins tied to the same IP or username.
    •             
  • Respond: Recommend temporary account lockout and IP blocking, then notify the               user.
    •             
  • Document: Record timeline, investigation steps, and outcome.
    •           
        
        
          
Case Study: Suspicious PowerShell Execution
          

            Scenario: Endpoint logs show unusual PowerShell usage on a user workstation.           

          
                
  • Review the exact command line and script content (if available).
    •             
  • Check parent processes, user context, and execution history.
    •             
  • Correlate with any network connections or file changes at that time.
    •             
  • Recommend isolating the host if behavior appears malicious.
    •             
  • Log findings and suggest additional hunting across similar endpoints.
    •
